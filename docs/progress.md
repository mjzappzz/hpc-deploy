# HPCDeploy 项目进度记录

> 维护流水与会话交接材料。它记录变化背景和待办，不替代 [architecture.md](architecture.md) 中的当前架构、安全边界或 [../deploy/README.md](../deploy/README.md) 中的可执行部署步骤。

## 当前完成度

### 2026-07-24 — SSH/SFTP 容错与控制台交互维护

- `SSHExecutor.connect()` 只建立 SSH 会话；命令 stdout 使用边界标记提取实际结果，远端 `.bashrc`、oneAPI、Lmod 等启动输出不再污染 HOME、PID、boot ID 和报告路径解析。
- SFTP 改为延迟创建；SFTP 子系统不可用或被 shell 输出破坏时，脚本上传自动回退到 SSH stdin 流式传输，结果目录扫描与 artifact 下载自动回退到 SSH 命令通道。
- boot ID 比较会从历史污染文本中提取 UUID，避免后端重启恢复监控时把同一台服务器误判为发生重启。
- 单任务取消使用独立的 120 秒前端超时；请求超时不再直接提示“取消任务失败”，而是提示后台可能仍在处理。
- 执行任务页的任务类型选择改为“环境部署 / 稳定性验证 / 资产分发”三列，列内任务卡片纵向排列，并在中等宽度和手机端分别降为两列、单列。
- 修复构建期传递依赖 `brace-expansion` 的安全公告，升级至 `2.1.2`，`npm audit` 为 0 vulnerabilities。
- 新增 shell banner、SFTP 上传 fallback、artifact 扫描/下载 fallback 与历史 boot ID 归一化回归测试。
- 批量部署公钥的未分类异常补充失败审计，记录目标服务器和错误上下文；单台失败继续不影响其他服务器。
- 系统设置拆分为轻量路由壳层和 `SettingsContent` 异步内容组件；“运行数据与路径”在接口完成前显示固定高度加载态，避免先显示错误的“缺失”状态及表格布局跳变。
- 管理区菜单改为显式路由选择；系统设置使用固定文本符号图标，规避部分浏览器在 SVG 图标重绘时出现闪烁、消失和悬停卡顿。
- 仪表盘“最近任务”新增独立任务 ID 列，统一复用历史任务的单次/批次与 GPU、CPU/内存、磁盘等标签规则；拖选任务名称或 ID 时不触发行跳转，正常单击仍进入对应单次或批次历史。

### 2026-07-23 — 执行任务步骤层级与 GPU 默认驱动

- 执行任务页将任务类型按“环境部署 / 稳定性验证 / 资产分发”分组展示；环境部署包含基础环境、GPU 驱动和 MPI 编译环境，压测与 Apptainer 分别独立成组。
- 四个执行步骤统一使用横向浅色标题带、左侧强调线和底部分隔线，完成状态与步骤操作保留在标题带中，强化长页面中的阶段边界。
- Linux NVIDIA 驱动库按类型自动选择默认版本：GeForce 使用 `580.159.04`，Data Center 使用 `580.173.02`；切换类型或切回驱动库来源时自动匹配，当前有效选择不会被覆盖。
- 驱动版本下拉项统一显示为“类型：完整文件名”，默认项追加“（默认）”，例如 `GeForce：NVIDIA-Linux-x86_64-580.159.04.run（默认）`。

### 2026-07-22 — 部署运行保障

- 修复 `sudo` 发布脚本丢失部署用户 NVM PATH、误用系统 Node.js 12 导致 Vite 构建失败的问题；安装与更新脚本现在统一解析 Node.js ≥18 的可用路径，构建失败仍发生在静态文件覆盖和服务重启之前。
- 补齐 FastAPI 文件上传运行依赖 `python-multipart`，避免新建后端隔离环境后因历史环境中的隐式依赖消失而启动失败。
- 安装与更新脚本在重启后端后等待 `127.0.0.1:8000/api/health` 检查通过，再重载 Nginx并报告成功，避免后端启动失败时产生发布成功假象。

### 2026-07-22 — Nginx 生产静态托管

- 前端由 Vite dev server 切换为 `npm run build` 后发布到 `/var/www/hpcdeploy`；为避开 Windows 宿主机现有 `80/tcp` 服务，生产 Nginx 统一监听 `10086/tcp`。
- Nginx 支持 Vue Router history fallback、`/api/` 反向代理、任务日志 WebSocket Upgrade、大文件上传和静态资源缓存。
- 旧 `hpcdeploy-frontend.service` 已移除；安装和更新脚本会清理遗留 unit。Vite 只保留本地开发命令，生产更新统一重新构建前端并重载 Nginx。
- 当前现场为 WSL NAT 环境，Windows 宿主机通过 `portproxy` 将 `10086/tcp` 同端口转发到 WSL，并配置入站防火墙规则；WSL IP变化后需更新转发目标。Linux 物理机/虚拟机直接部署时不需要该 Windows 转发步骤。

### 2026-07-22 — 新机 Linux 基础策略脚本

- 任务执行的业务类型拆分为基础环境配置、GPU 驱动安装、MPI 编译环境配置、Linux 服务器压测和 Apptainer 镜像；后端仍保持 `script/stress/apptainer` 任务契约与 GPU/CUDA 专用接口。
- 基础环境配置与 GPU 驱动安装支持多选：前者固定按关闭锁屏/休眠 → 锁定系统版本，后者固定按 NVIDIA 驱动 → CUDA Toolkit 串行执行；同服务器前序失败时不启动后序任务，多服务器之间并行。
- 资产库同步按基础环境配置、MPI 编译环境配置、Linux 服务器压测和 Apptainer 分类筛选；基础环境与 MPI 编译环境脚本继续使用同一受控 `backend/scripts/mpi/` 物理目录。
- 服务器环境新增“Linux 系统版本锁定”与“Linux 关闭锁屏与休眠”两张独立卡片，继续通过白名单脚本任务执行。
- 版本锁定在 Rocky Linux 9.4 上使用 Vault 并锁定 release/repo/GPG 包，其他 Rocky 版本直接跳过；Ubuntu 22.04/24.04 禁止发行版跨版升级。脚本不执行全量系统升级，也不做跨版本降级。
- 关闭锁屏与休眠脚本配置 systemd/logind、GNOME dconf、KDE 和 X11 autostart；不立即重启 `systemd-logind`，避免中断当前会话。
- 两个脚本都要求 root，修改前备份到 `/var/backups/hpcdeploy/`，并在执行后输出 PASS/FAIL 验证结果。
- 普通脚本非零退出时，任务失败原因优先提取最近的 `[ERROR]` 具体错误；只有脚本未输出具体错误时才回退到 `command exited with code N`。
- 历史任务卡片统一展示任务/子任务名称、脚本文件、任务状态和详情说明；成功跳过、失败与取消均可直接查看具体说明，执行命令保留在单次和批次详情中。
- 历史任务默认每页展示 20 条；非压测任务隐藏“结果文件”，压测批次中的 XLSX 最终报告使用独立高亮样式。
- CUDA 安装成功后，单次任务及包含 CUDA 子任务的批次卡片均支持从任务日志复制环境变量和验证命令。
- 删除任务入口固定在任务卡片右下角，默认显示小红色三角标；右下角保留原按钮大小的透明触发区，悬停或键盘聚焦时以轻滑、渐显和阴影动画展开完整删除按钮，并继续要求管理员确认。

### 2026-07-17 — 管理员模式、任务清理与执行页复检

- 修复后端重启后的 stress 恢复：恢复监控 SSH 临时不可用时，先重试 3 次；仍不可用则保留任务 `RUNNING` 并在 60 秒后继续恢复，避免 detached 的远端压测被误标为失败。诊断规则优先识别恢复 SSH 错误，不再将正常的 GPU/CUDA 启动日志误判为 GPU 环境异常。
- 单次任务详情的 GPU 面板与批次子任务统一为同一组紧凑卡片：显卡序号、利用率、显存、温度、风扇、功耗、P-State 与 PCIe Bus-ID 使用一致的布局和字段顺序。
- 历史任务的单次抽屉与批次详情 GPU 面板补齐远端 `nvidia-smi` 运行指标：风扇转速、实时/上限功耗、性能状态和 PCIe Bus-ID；接口采用可选新增字段，保持旧客户端兼容。
- 顶部新增普通/管理员模式切换。管理员会话有效期为 5 分钟，显示倒计时；使用 HttpOnly Cookie 支持刷新恢复，手动退出或超时后保持普通模式。
- 普通模式保留新增/编辑服务器、部署公钥、上传脚本密码确认、提交/取消任务等日常操作；管理员模式负责删除与系统设置类高风险操作。
- 审计日志在管理员模式下直接加载；普通模式点击侧栏入口显示权限提示。
