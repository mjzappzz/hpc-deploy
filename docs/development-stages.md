# HPCDeploy 阶段开发计划

版本：v2.1
用途：给后续开发按阶段推进使用
原则：先收敛主线，再逐步扩展；每个阶段只解决当前明确范围内的问题。

---

## 0. 当前主线

HPCDeploy 已完成完整 MVP 链路：

```text
服务器管理 → 脚本知识库 → 任务执行 → SSH 上传 → 远程执行
→ 实时日志 → 资源快照 → 任务历史 → 结果文件回收
→ 取消任务 → 删除任务 → 仪表盘总览 → 目录树查看
```

四类知识库目录：

```text
backend/scripts/mpi       # 编译环境
backend/scripts/stress    # 压测脚本
backend/scripts/test      # 测试脚本
backend/apptainer         # 容器文件（仅分发）
```

---

## 1. 开发总原则

```text
先完成脚本知识库收敛 → 再简化任务执行页 → 再补真实远程执行
→ 再做结果回收 → 最后补实时日志、并发控制、部署强化和仪表盘
```

执行原则：

```text
1. 不做复杂编排
2. 不做 AI 助手能力
3. 不做通用参数 Schema 平台
4. 不做 WebSocket（用 HTTP 轮询替代）
5. 不做 Server Lock（用简单防重复提交替代）
6. 所有新增功能优先服务核心主链路
7. 安全优先：白名单执行、前端不传命令、路径防逃逸
```

---

## 4. 禁止事项

```text
1. 禁止跳阶段一次性做完全部功能
2. 禁止前端直接执行 SSH
3. 禁止后端执行前端传入的任意命令
4. 禁止绕过脚本白名单
5. 禁止把 SSH 私钥返回给前端
6. 禁止没有日志的后台任务
7. 禁止任务没有状态
8. 禁止同一服务器无控制地并发安装
9. 禁止 apptainer run/exec
```

---

## 5. 最终原则

```text
先跑通主链路，再优化 UI，再扩展功能。
```

主链路：

```text
服务器 → 脚本 → 参数 → SSH 执行 → 实时日志 → 任务记录 → 结果回收
```
