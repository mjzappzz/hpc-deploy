# 趣味性设计原则

> 生效日期：2026-07-08

## 原则

HPCDeploy 是一个正经的运维工具，但**正经 ≠ 死板**。在不影响功能使用、不增加资源开销的前提下，适当加入趣味元素，让运维工作不那么枯燥。

## 适用范围

所有用户可见的界面层：按钮文案、加载提示、空状态、小交互反馈。

## 规则

### 1. 搞怪但不碍事

- 趣味元素**不能阻挡或干扰**正常操作路径
- 不能导致操作延迟或卡顿
- 不能遮挡关键信息

### 2. 低资源开销

- 趣味元素优先使用 **CSS 动画 + Unicode 字符**，禁止引入额外图片/库
- 禁止使用大体积资源（gif、视频、第三方动画库）
- 定时器只用于煤球生命周期、发言和卡片观察；离开视口或卸载时必须清理

### 3. 适可而止

- 同一个用户会话中，相同趣味文案不重复出现
- 错误/警告场景不要开玩笑（如 "SSH 连接失败" 时保持严肃）
- 用户连续操作时不刷屏

### 4. 有退路

- `prefers-reduced-motion` 媒体查询降级：检测到用户开启系统减少动效时，所有动画失效
- 所有趣味文案不影响国际化（目前无 i18n，后续如需则统一管理）

## 已实现

| 位置 | 类型 | 实现方式 |
|------|------|----------|
| 卡片边缘小煤球 | CSS 动效 | `AppCritters.vue` 全局 IntersectionObserver 驱动，卡片进入视口时生一只 |
| 检测加载文案 | 随机文案 | `useFunMessages.ts` 吐槽池，SSH 检测时随机抽取 |
| 空状态文案 | 趣味文案 | 各页面 `el-empty` description 替换为搞怪文本 |

## 煤球设计规范（AppCritters.vue）

- **配色**：纯黑色系（`#1a1a1a` ~ `#0d0d0d`），不许五颜六色
- **造型**：圆形毛绒煤球、白眼黑瞳与少量小脚；不使用手、角或天线。眼睛和小脚随所在边缘镜像，始终朝向卡片中心
- **语言**：全中文吐槽、运维感悟与轻量对话；不使用固定入场问候或离场告别
- **行为**：每只独立 26~42 秒生命周期，静默进出；入场完成后即开始随机发言，单句显示约 4.2~5.4 秒，句间停顿约 2.5~4 秒
- **命名**：`小黑子1号`~`小黑子10号` 与煤球球、黑子、碳碳、团团等名称随机出现，同屏不重名；不使用“墨墨”
- **实现**：DOM 注入 + CSS animation + IntersectionObserver，零图片

## 煤球交互与可读性

- **投放范围**：仅主内容区的普通业务卡片；弹窗、抽屉、通知和错误/告警卡片不生成煤球。
- **数量与生命周期**：全局最多 2 只、同一卡片最多 1 只；煤球以卡片边缘探头的方式出现，每只独立存活 26~42 秒。仅保留一个 20 秒保底定时器，动态卡片由 `MutationObserver` 发现。
- **独白与对话**：单只煤球持续随机发言；同一卡片距离足够近的两只煤球可轮流对话。文案全局保留最近 24 条历史，优先去重。
- **文案池**：普通煤球使用 300 条共享池；小黑子使用 100 条专属阴阳吐槽池，普通煤球不抽取该专属池。
- **定位与避让**：煤球本体为 11px 边缘探头，眼睛保持清晰可辨；彼此至少间隔 120px。气泡沿煤球朝卡片中心的方向展开，最大宽度 260px、相邻时 220px，最多两行。左右名称位于卡片外侧并竖排；气泡仅在不覆盖卡片区域时显示。
- **动效与无障碍**：煤球本体使用平滑浮动与淡入淡出；名字保持静止，避免文字抖动。系统开启 `prefers-reduced-motion` 时不生成煤球。

## 趣味文案池位置

- 煤球独白/对话池：`frontend/src/components/AppCritters.vue` 中的 `SARCASTIC_MESSAGES`、`SHARED_MESSAGES`、`PERSONAS`、`PAGE_SOLO_MESSAGES`、`PAIR_DIALOGUES`、`LIFE_DIALOGUES`
- 运维吐槽（加载文案）：`frontend/src/composables/useFunMessages.ts`
