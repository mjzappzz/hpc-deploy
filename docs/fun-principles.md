# 趣味性设计原则

> 生效日期：2026-07-08

## 原则

HPCDeploy 是一个正经的运维工具，但**正经 ≠ 死板**。在不影响功能使用、不增加资源开销的前提下，适当加入趣味元素，让运维工作不那么枯燥。

## 适用范围

所有用户可见的界面层：按钮文案、加载提示、空状态、小交互反馈。

## 规则

### 1. 搞怪但不碍事

- 趣味元素**不能阻挡或干扰**正常操作路径
- 不能导致操作延迟或卡顿
- 不能遮挡关键信息

### 2. 零资源开销

- 趣味元素优先使用 **CSS 动画 + Unicode 字符**，禁止引入额外图片/库
- 禁止使用大体积资源（gif、视频、第三方动画库）
- JS 定时器控制在每个页面 ≤ 1 个，间隔 ≥ 15s

### 3. 适可而止

- 同一个用户会话中，相同趣味文案不重复出现
- 错误/警告场景不要开玩笑（如 "SSH 连接失败" 时保持严肃）
- 用户连续操作时不刷屏

### 4. 有退路

- `prefers-reduced-motion` 媒体查询降级：检测到用户开启系统减少动效时，所有动画失效
- 所有趣味文案不影响国际化（目前无 i18n，后续如需则统一管理）

## 已实现

| 位置 | 类型 | 实现方式 |
|------|------|----------|
| 卡片边缘小煤球 | CSS 动效 | `AppCritters.vue` 全局 IntersectionObserver 驱动，卡片进入视口时生一只 |
| 检测加载文案 | 随机文案 | `useFunMessages.ts` 吐槽池，SSH 检测时随机抽取 |
| 空状态文案 | 趣味文案 | 各页面 `el-empty` description 替换为搞怪文本 |

## 煤球设计规范（AppCritters.vue）

- **配色**：纯黑色系（`#1a1a1a` ~ `#0d0d0d`），不许五颜六色
- **眼睛**：白色大圆 + 黑色小瞳孔，不同边不同位置（上下并排、左右竖排）；本体保持柔和黑灰渐变
- **语言**：全中文吐槽（"好闲啊…"、"又在摸鱼"、"什么时候下班"等）
- **行为**：每只独立 26~42 秒生命周期，出现自我介绍、存活期间自动吐槽、消失说再见
- **命名**：小黑子、煤球球、黑子、碳碳、团团等煤球系名字随机；不使用“墨墨”
- **实现**：DOM 注入 + CSS animation + IntersectionObserver，零图片

## 煤球交互与可读性

- **投放范围**：仅主内容区的普通业务卡片；弹窗、抽屉、通知和错误/告警卡片不生成煤球。
- **数量与生命周期**：全局最多 3 只，每只独立存活 26~42 秒；仅保留一个 15 秒保底定时器，动态卡片由 `MutationObserver` 发现。
- **独白与对话**：单只煤球独立吐槽；同一卡片的两只煤球只有在距离足够近时才轮流对话，距离较远则各自独白。对话按当前页面（仪表盘、服务器、任务执行、历史、脚本、设置）选择文案，并带全局冷却避免刷屏。
- **定位**：名字和气泡均沿煤球朝卡片中心的方向展开；气泡尾巴指向煤球，默认单行展示，空间不足时最多两行。
- **动效与无障碍**：煤球本体、名字和气泡使用整数像素步进浮动，避免小字号文字发虚；系统开启 `prefers-reduced-motion` 时不生成煤球。

## 趣味文案池位置

- 煤球独白/对话池：`frontend/src/components/AppCritters.vue` 中的 `PERSONAS`、`PAGE_SOLO_MESSAGES`、`PAIR_DIALOGUES`、`LIFE_DIALOGUES`
- 运维吐槽（加载文案）：`frontend/src/composables/useFunMessages.ts`
